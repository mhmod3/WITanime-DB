# WITanime DB

قاعدة بيانات مفتوحة المصدر لتحديث روابط البث والتحميل لحلقات الأنمي. تُخزّن البيانات في ملفات `JSON` منفصلة لكل أنمي، مع تحديث دوري آلي لضمان صلاحية الروابط.

## هيكل المشروع

    WITanime-DB/
    ├── data/
    │   ├── Gamers.json
    │   ├── AttackOnTitan.json
    │   └── ...
    ├── scripts/
    │   └── updater.py
    └── README.md

## نموذج ملف JSON

    {
        "!Gamers": {
            "anime_url": "https://witanime.you/anime/gamers/",
            "type": "TV",
            "mal_id": "34280"
        }
    }

- **المفتاح الرئيسي**: اسم الأنمي (مسبوق بـ `!` لتحديد الأولوية).
- **anime_url**: الرابط الأساسي للأنمي على المصدر.
- **type**: نوع العرض (TV, Movie, OVA, إلخ).
- **mal_id**: المعرف الرسمي من MyAnimeList.

## آلية التحديث الدوري

يُشغّل سكريبت آلي يوميًا (عبر GitHub Actions) لفحص الروابط واستبدال التالفة منها، مع تسجيل تغييرات الإصدارات في ملف `changelog`.

# الأسئلة الشائعة (FAQ)

<details>
<summary><strong>١. ما هو الغرض الأساسي من المشروع؟</strong></summary>

توفير مصدر موثوق ومُحدّث تلقائيًا لروابط مشاهدة وتحميل حلقات الأنمي، لتسهيل دمجها في تطبيقات الطرف الثالث أو مواقع المعجبين دون الحاجة لفحص يدوي.
</details>

<details>
<summary><strong>٢. كيف أعرف أن الرابط لا يزال صالحًا؟</strong></summary>

يحتوي السكريبت على اختبار استجابة (HTTP status check) لكل رابط. إذا عاد بـ 404 أو 503، يُعلّم الرابط كـ `"expired": true` في الإصدار التالي من الملف.
</details>

<details>
<summary><strong>٣. هل يمكن إضافة أنمي جديد يدويًا؟</strong></summary>

نعم. أضف ملف JSON جديد في مجلد `data/` بنفس البنية، ثم أرسل طلب سحب (Pull Request). سيتولى السكريبت التحقق من البنية وتحديث الروابط تلقائيًا في الدورة التالية.
</details>

<details>
<summary><strong>٤. ما هو تنسيق تاريخ التحديث في الملفات؟</strong></summary>

يُضاف حقل `"last_updated"` بصيغة ISO 8601 (مثل `2026-06-16T10:30:00Z`) إلى جذر كل ملف JSON عند كل تعديل ناجح.
</details>

<details>
<summary><strong>٥. هل تدعم المشروع روابط التحميل المباشر أم روابط البث فقط؟</strong></summary>

كليهما. يفترض أن يحتوي `anime_url` على صفحة تضم كلا الخيارين. في الإصدارات المستقبلية سيُفصل كل رابط في حقلين منفصلين (`stream_url`, `download_url`) عند توفر البيانات.
</details>

<details>
<summary><strong>٦. كيف يمكنني تشغيل سكريبت التحديث محليًا؟</strong></summary>

    pip install -r requirements.txt
    python scripts/updater.py --path data/

يقرأ السكريبت جميع ملفات JSON، ويجلب الروابط الجديدة، ويعيد الكتابة فوق الملفات القديمة مع الاحتفاظ بسجل التغييرات في مجلد `logs/`.
</details>

<summary><strong>٧. ما هي رخصة المشروع؟</strong></summary>

<summary><strong>٨. كيف أبلغ عن رابط معطل أو خطأ في التصنيف؟</strong></summary>

افتح Issue في GitHub مع ذكر اسم الأنمي وقيمة `mal_id`، وسيُعالَج خلال دورة التحديث التالية (أقل من ٢٤ ساعة).
</details>
